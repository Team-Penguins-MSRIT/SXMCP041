It detects **Bus Factor risk** from complex system commit logs, analyzes key-person dependencies utilizing a swarm of specific MCPs (Model Context Protocol), and securely dispatches a polished, enterprise-grade PDF risk report directly to a decision-maker\'s WhatsApp.\n\n### 🌟 Key Features\n\n- **Commit Log Ingestion:** Parses raw repository histories.\n- **Risk Knowledge Graph:** Dynamically models developer dependencies.\n- **Bus Factor Detection:** Identifies critical key-person concentration risks.\n- **Autonomous Swarm:** Orchestrates sequential-planning across 5 distinct MCP servers.\n- **Automated Reporting:** Generates professional PII-scrubbed PDF reports.\n- **WhatsApp Bridge Integration:** Securely delivers reports directly to user devices via a local WhatsApp MCP instance.\n\n---\n\n## 🛠️ Architecture\n\nThe orchestrator boots **5 distinct MCP servers** to handle the pipeline:\n1. **WhatsApp Bridge (SSE)** - Secure document and message delivery.\n2. **Custom Analysis (stdio)** - Analyzes commits and runs metrics calculations.\n3. **Sequential Thinking (stdio)** - Plans and reasons through the M&A evaluation.\n4. **Memory / KG (stdio)** - Maintains a knowledge graph of contributors and files.\n5. **Filesystem (stdio)** - Read/write access for cache files and raw evidence.\n\n---\n\n## 🚀 Step-by-Step Setup\n\nFollow these instructions to get KEYSTONE running on your local machine.\n\n### 1. Prerequisites\n- **OS:** Windows + PowerShell (or Unix-based)\n- **Python:** 3.11+\n- **Node.js:** v18+ & `npx`\n- **Docker Desktop:** Required for the WhatsApp MCP bridge\n- **Keys:** Groq API key (`GROQ_API_KEY`)\n\n### 2. Environment Setup\n\nClone the repository and set up your Python virtual environment:\n\n```powershell\n# Create active virtual environment\npython -m venv venv\n.\\venv\\Scripts\\activate\n\n# Install core requirements\npip install -r requirements.txt\n\n# Install specific NLP dependencies\npip install fpdf2 presidio-analyzer presidio-anonymizer spacy\npython -m spacy download en_core_web_lg\n```\n\n### 3. Configure API Keys\n\nCreate a `.env` file in the root of the repository:\n\n```env\nGROQ_API_KEY=your_groq_key_here\nTARGET_PHONE=91XXXXXXXXXX\nWHATSAPP_BRIDGE_API_KEY=generate_some_secure_string_here\n```\n\n### 4. Start the WhatsApp MCP Stack\n\nSpin up the Docker containers required for the WhatsApp Bridge:\n\n```powershell\ncd mcp-servers\\whatsapp-mcp-extended\n# Start the containers in detached mode\ndocker-compose up -d\n```\n> **Note:** Ensure your WhatsApp bridge is connected, authenticated via the QR code on the Web UI (port 8091 by default), and actively running.\n\n### 5. Frontend Dashboard (Optional)\n\nKEYSTONE has a tailored dashboard available in the `keystone` folder:\n\n```powershell\ncd keystone\nnpm install\nnpm run dev\n```\n\n### 6. Run the Swarm Orchestrator\n\nReturn to the root directory and initiate the due diligence pipeline:\n\n```powershell\npython main.py\n```\n\nAlternately, start the FastAPI server to use the web dashboard endpoints:\n\n```powershell\npython api.py\n# or\nuvicorn api:app --reload\n```\n\n---\n\n## 📂 Repository Layout\n\n- `main.py` - Core swarm orchestrator CLI.\n- `api.py` - REST & SSE API for the visual frontend.\n- `custom_tools.py` - FastMCP server for log ingestion and macro analysis.\n- `bus-factor-detector/` - Analyzer code, raw commit samples, and evidence cache.\n- `mcp-servers/whatsapp-mcp-extended/` - Dockerized Go bridge and Python MCP WhatsApp tools.\n- `keystone/` - React/Vite interactive frontend for viewing the generated Knowledge Graphs.\n\n---\n\n## 📈 Outputs & Artifacts\n\n- **PDF Reports:** Generated and saved automatically in `mcp-servers/whatsapp-mcp-extended/store`.\n- **Macro Cache:** Processed analytics saved as JSON in `bus-factor-detector/cache/macro_result.json`.\n- **Evidence:** Extracted log details in `bus-factor-detector/evidence/`.\n\n---\n\n<p align="center">\n  Built with ❤️ by <b>Team Penguins - MSRIT</b>\n</p>
